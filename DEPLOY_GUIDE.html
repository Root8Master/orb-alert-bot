1

Create Upstox Developer App

  1. Go to developer.upstox.com → login with Upstox credentials
  2. Click My Apps → New App
  3. Fill in:
    • App Name: ORB Alert Bot
    • Redirect URL: https://orb-alert-bot.onrender.com/callback
      (Update after Step 5 once you know your Render URL)
  4. Click Create
  5. Note your API Key → this is client_id in config.ini
  6. Note your API Secret → this is client_secret in config.ini
⚠️ Make sure NSE EQ segment is active in your Upstox account. Open Upstox app → Profile → Segments.
ℹ️
Why does the bot need my password, PIN, and TOTP secret?
Upstox does not provide a server-to-server token refresh API. Every day the access token expires and the only way to get a new one is to complete the full login flow: mobile → password → PIN → TOTP code. The upstox-totp library automates this headlessly at 08:50 AM IST so you never have to do it manually. All 5 credentials are required by the library — remove any one and login fails.

These values are stored as Render environment variables and are never written to disk, logged, or pushed to GitHub.
2

Get Your TOTP Secret Key

This lets the bot auto-login every day at 08:50 IST — no manual action ever.

  1. Open Upstox app on phone → Profile → Settings → Two-Factor Authentication
  2. If TOTP not enabled → enable it. The setup screen shows a QR code + secret key.
  3. If already enabled with Google Authenticator:
    • Disable TOTP → re-enable
    • On setup screen tap "Can't scan QR?" or "Show key"
  4. Copy the alphanumeric string — looks like JBSWY3DPEHPK3PXP
  5. Also scan into Google Authenticator normally — you still need it to log in manually
⚠️ This is a one-time step. Save the key somewhere secure. You will never need to do this again.
3

Create Telegram Bot

  1. Open Telegram → search @BotFather → send /newbot
  2. Name: ORB Alert Bot
  3. Username: anything ending in bot e.g. orb_nifty_alert_bot
  4. BotFather replies with token: 7123456789:AAFxxxxxxxxxxxxxxx → this is bot_token
  5. Search @userinfobot → send /start → copy the Id: number → this is chat_id
  6. Search your bot name → press Start (required — bot cannot message you until you do this)
💡 Want alerts in a group? Add the bot to a Telegram group, then use the group's chat ID as chat_id.

5

Setup Google Sheets (Paper Trade Log)

Trades are logged to a Google Sheet automatically — visible in real-time, no database needed.

5a. Create the Google Sheet

  1. Go to sheets.google.com → Create new spreadsheet
  2. Name it: ORB Trade Log
  3. Copy the Sheet ID from the URL:
    https://docs.google.com/spreadsheets/d/THIS_PART/edit
    That highlighted part is your sheet_id in config.ini

5b. Create a Google Cloud Service Account

  1. Go to console.cloud.google.com → Login
  2. Create a new project → name it ORB Bot
  3. Search for Google Sheets API → Enable it
  4. Go to IAM & Admin → Service Accounts → Create Service Account
  5. Name: orb-bot → click Create and Continue → Done
  6. Click the service account → Keys → Add Key → Create New Key → JSON
  7. A .json file downloads automatically — save it as service_account.json inside the orb_alert folder

5c. Share the Sheet with the Service Account

  1. Open your Google Sheet
  2. Click Share (top right)
  3. Open service_account.json — find the "client_email" field
    Looks like: orb-bot@orb-bot-123456.iam.gserviceaccount.com
  4. Paste that email into the Share dialog → Role: Editor → Share

5d. Update config.ini

[google]
sheet_id             = 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms   # from Sheet URL
service_account_json = service_account.json   # file in orb_alert folder
✅ setup.py reads the JSON file and pushes it as a single env var to Render automatically — you never paste JSON manually.
ℹ️ The bot auto-creates a Trades sheet tab with headers on first run. Sheet columns: ID · Date · Time · Symbol · Index · Entry · SL · Target · Exit Price · Exit Reason · Exit Time · P&L% · Status

  3. Settings:

  4. Click Create Web Service
  5. Note your Render URL e.g. orb-alert-bot.onrender.com

5b. Get Render API Key + Service ID

ItemWhere to find
API KeyRender → Account Settings → API Keys → Create API Key

Go back to developer.upstox.com → My Apps → Edit
Update Redirect URL to: https://orb-alert-bot.onrender.com/callback

7

Fill config.ini and Run setup.py

Open config.ini in any text editor. Fill in all values:

# ── config.ini — fill all values, then run setup.py ────

[upstox]
mobile        = 9876543210         # your 10-digit mobile
password      = MyPassword123      # Upstox login password
pin           = 654321             # 6-digit Upstox PIN
totp_secret   = JBSWY3DPEHPK3PXP   # from Step 2
client_id     = abc123xyz          # API Key from Step 1
client_secret = def456uvw          # API Secret from Step 1

[telegram]
bot_token     = 7123456789:AAFxxx... # from Step 3
chat_id       = 987654321           # from Step 3

[render]
api_key       = rnd_xxxxx...        # Render → Account Settings → API Keys
service_id    = srv_xxxxx...        # Render → your service URL → srv_xxx part

[google]
sheet_id             = 1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms   # from Sheet URL (Step 5)
service_account_json = service_account.json                             # file in orb_alert folder

[scanner]
interval_seconds  = 300
market_open_hhmm  = 09:15
market_close_hhmm = 11:00
heartbeat_minutes = 60

[paper_trades]
sl_pct   = 0.008   # 0.8% stop loss
tgt_pct  = 0.015   # 1.5% target

Then run in PowerShell from inside the orb_alert folder:

pip install requests
python setup.py

The script will:

  1. Validate all fields are filled
  2. Verify Telegram bot token is valid
  3. Push all 19 env vars to Render via API
  4. Trigger a redeploy
  5. Print your service URL
✅ You should see Telegram bot OK and All env vars set on Render in the output. If not, fix the error shown before continuing.

Troubleshooting

setup.py fails validation
Placeholder values still in config.ini. Replace all your_xxx values.
Telegram bot invalid
Double-check token — no spaces, must include the colon. Start the bot first.
Render 401 / 404
API key or service_id wrong. Regenerate API key in Render settings.
Token refresh fails at 08:50
TOTP secret wrong. Must be raw base32 key — not a QR image, not a URL.
No alerts during 09:20–11:00
Use POST /run-now to trigger manually and check GET / for errors.
All stocks skipped by sector filter
Broad market red day. Expected. All sector indexes are down — scanner correctly silent.
Render keeps sleeping
Check UptimeRobot monitor is active and hitting /health every 5 min.
Trades not appearing in Sheet
Sheet not shared with service account email. Open Sheet → Share → paste client_email from JSON → Editor role.
gspread auth error on startup
GOOGLE_SERVICE_ACCOUNT_JSON env var malformed. Re-run setup.py — it reads the file and pushes it correctly.
Google Sheets API not enabled
Go to console.cloud.google.com → your project → APIs → enable Google Sheets API.
NSE EQ segment error
Open Upstox app → Profile → Segments → activate NSE EQ.
⚠️ Watchdog alert fires every day
Scanner not running by 09:30. Check token refresh at 08:50 — likely TOTP secret wrong or Upstox login failing.
Bot scanning on market holiday
Holiday not in hardcoded list. Check GET /holiday — if wrong, NSE API fetch may have failed. Check Render logs.
Same stock alerting twice after restart
Dedup Sheet tab missing. Check Google Sheet has a Dedup tab. Bot creates it automatically on first run — verify Sheets access is working.
Telegram commands not responding
Did you send /start to the bot? Also verify TELEGRAM_CHAT_ID matches your actual chat ID — bot ignores other chat IDs for security.
Rate limit errors in Render logs
Expected on market open — 62 stocks × 2 API calls = 124 calls/cycle. Exponential backoff handles it automatically. Errors resolve within 8–16 seconds.
